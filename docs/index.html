An entropy-guided scaling framework for predicting recommendation performance across model, data, and training settings.

1. Method: From Scaling Law to P-Law

P-Law predicts large recommendation model performance through ranking metrics such as HR and NDCG, using entropy-guided data quality and a fitted loss-performance discrepancy term.

1. Measure Entropy-Guided Data Quality

Estimate sequence complexity so redundant, low-information interactions have less influence than raw dataset size.

2. Fit Quantitative Performance Surfaces

Fit HR/NDCG over model depth, embedding dimension, data quality, and training settings.

3. Predict Scaling Decisions

Use fitted surfaces to estimate useful parameter choices and model-expansion potential without exhaustive full-scale training.

2. Code

The repository contains an entropy utility, transformer-based recommendation training code, and appendix scripts for P-Law fitting and visualization.

pip install -e PerformanceLaw
Train a transformer recommender 
cd General_Transformer
torchrun --standalone --nproc_per_node=1 main.py --model_name llama --n_layers 8
Fit appendix performance laws 
cd Performance_Law_Appendix_Result
python performance_law_fitting.py

3. Experimental Highlights

The fitted surfaces show that recommendation performance depends jointly on model scale, data quality, and the gap between optimization loss and ranking performance.

HR
Direct performance modeling

The framework fits ranking performance directly rather than only modeling training loss.

Entropy
Data quality signal

Entropy guidance captures sequence complexity and reduces the effect of redundant interaction histories.

U-shape
Configuration insight

Performance can improve, saturate, and decay as model, data, and training settings change.
